1 $PATH, переменные окружения, как изменить поведение терминала

Может вы еще не знаете, но терминал в Linux - это диалог с системой программирования на языке Bash. В нем можно писать циклы и определять переменные - с тем, чтобы писать скрипты.

Когда вы запустили терминал, в нем уже определены некоторые переменные. Эти переменные определяют поведение системы. Сейчас нам нужно изменить переменную PATH. В этой переменной хранится список директорий, где система будет искать программы. Когда вы набираете firefo и жмете следом TAB, терминал дополняет firefo до слова firefox, поскольку знает, что первым словом указывается название программы, а программа firefox лежит не где-то в неизвестной директории, а в директории, которая есть в переменной PATH.

Вы можете посмотреть значение переменной PATH: 

echo $PATH
# /run/wrappers/bin:/run/current-system/sw/bin

На прошлом семинаре мы поставили dat с помощью npm в директорию ~/.npm-packages/bin. Нам пришлось настроить npm, сделав правку в файле ~/.npmrc. Многие заметили, что система не способна найти программу dat: 

dat
# sh: line 1: dat: command not found

Чтобы запустить dat, приходилось указывать полный путь к программе от домашней директории: 

~/.npm-packages/bin/dat
# Usage: dat <cmd> [<dir>] [options]
# 
# Sharing Files:
#    dat share                   create dat, import files, share to network
#    dat create                  create empty dat and dat.json
#    dat sync                    import files to existing dat & sync with network

Естественное желание - изменить переменную PATH так, чтобы терминал автоматически xнаходил команду dat.

Кратковременный эффект можно получить, выполнив в терминале: 

export PATH=$HOME/.npm-packages:$PATH

Попробуйте выполнить следующее упражнение, чтобы понять, как работает вышеописанный кусок кода.

Вы обратили внимание на переменную HOME? Посмотрите, чему равно ее значение. Определите значение переменной QWER равное "world". Проверьте значение QWER. Выполните: 

export QWER="hello "$QWER

Посмотрите теперь на значение QWER.

Теперь dat можно запустить, набрав в этом же терминале просто dat. Однако при открытии другого терминала команда dat не сработает и вам снова придется переопределять переменную PATH. Почему? Оказывается, каждый раз при запуске терминала система пытается выполнить программу на языке bash в файлике ~/.bashrc. Правка этого файла может коренным образом поменять поведение терминала, в том числе поможет избавиться от постоянных переопределений переменной PATH. Добавьте в этот файл строку: 

export PATH=$HOME/.npm-packages:$PATH

Теперь в любом терминале вы сможете быстро запустить dat.
