Конфигурация emacs лежит в файле: ~/.emacs. Открыть файл можно с помощью любого текстового редактора.

Ниже приведен пример того, как должен выглядеть минимальный конфиг для этого семинара:

;; installed packages.  Don't delete this line.  If you don't want it,
;; just comment it out by adding a semicolon to the start of the line.
;; You may delete these explanatory comments.

(package-initialize)

;; (setq inhibit-startup-message t)
;; (tool-bar-mode -1)

(require 'package)
(setq package-enable-at-startup nil)
(add-to-list 'package-archives '("melpa" . "http://melpa.org/packages/"))

(unless (package-installed-p 'use-package)
  (package-refresh-contents)
  (package-install 'use-package))

(use-package try
  :ensure t)

(use-package htmlize
  :ensure t)

4 Dat protocol. Как обмениваться файлами без облаков.

4.1 Что это такое

У вас много файликов на компе и надо пошарить директорию среди друзей без третьей стороны.

4.2 Какие есть решения

  1. ftp сервер и web сервер
  2. облачные хранилища - вот так нельзя!
  3. torrent - хороший инструмент, но в нем нет возможности синхронизовывать обновления директории
  4. dat протокол

4.3 Установка dat через npm

Так обычно делают javascript программисты.

4.3.1 Нужен nodejs

У обладателей NixOS он уже стоит.

4.3.2 Нужен npm - менеджер пакетов

nodejs тянет за собой npm.

4.3.3 Настройка npm

Надо сделать так, чтобы npm ставил программы в определенную директорию.

Для этого надо отредактировать файлик ~/.npmrc: 

prefix = ${HOME}/.npm-packages

4.3.4 Ставим dat командой

npm install -g dat

4.4 Используем dat

Сценарий: один желает делиться контентом некой директории foo, остальные желают иметь свежую версию этого контента.

4.4.1 Желающий поделиться

Идет в директорию foo и запускает там dat: 

cd foo
~/.npm-packages/bin/dat share

После запуска dat сгенерирует ссылку, по которой будет доступен foo.

4.4.2 Желающий получить

Желающий получить foo на своей машине выполняет: 

~/.npm-packages/bin/dat clone <dat ссылка> foo

Возможно dat clone не с первого раза найдет информацию по dat ссылке.

После удачного клонирования в директорию foo вы можете выбрать роль раздающего этот контент. Для этого надо перейти в foo и запустить там dat sync: 

cd foo
~/.npm-packages/bin/dat sync
